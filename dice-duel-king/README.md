- Tracking issue: https://github.com/weidonglang/game/issues/1

## Core Rules Implemented

- The player and the computer each start with 30 HP.
- Every round both sides choose one of four skills:
  - Steady Strike: roll `2d6` for reliable damage.
  - Heavy Blow: roll `1d20`; values below 6 miss, high values hit hard.
  - Guard: roll `1d8` for block and deal a small counterattack.
  - Charge: gain stored power and deal light chip damage.
- Guard reduces incoming damage for that round.
- Charge adds bonus dice to the next attacking skill.
- The computer uses a simple state-based strategy instead of pure random play.
- Each completed game appends round-by-round results to `data/duel_log.csv`.

## How To Run

### Build

```powershell
cd E:\javacode\game-automation-repo\dice-duel-king
.\build.bat
```

Manual build:

```powershell
cd E:\javacode\game-automation-repo\dice-duel-king
gcc .\src\main.c -o .\dice_duel.exe
```

### Play

```powershell
cd E:\javacode\game-automation-repo\dice-duel-king
.\dice_duel.exe
```

### Run Non-Interactive Verification

```powershell
cd E:\javacode\game-automation-repo\dice-duel-king
.\dice_duel.exe --self-test
```

### Generate R Simulation

If `Rscript` is installed and on `PATH`:

```powershell
cd E:\javacode\game-automation-repo
Rscript .\dice-duel-king\analysis\simulate.R
```

The R script simulates 10000 games and writes:

- `dice-duel-king/data/strategy_summary.csv`
- `dice-duel-king/analysis/strategy_win_rates.png`

## Project Files

- `src/main.c`: turn loop, dice skills, computer strategy, CSV logging, self-test
- `build.bat`: local GCC build helper
- `analysis/simulate.R`: base R strategy simulator and win-rate plotter
- `data/duel_log.csv`: generated by played or self-test games
